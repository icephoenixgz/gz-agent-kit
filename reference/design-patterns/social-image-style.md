# 社交配图风格定义参考

> 来源于 [SpaceZephyr/space-GPT-image2-design](https://github.com/SpaceZephyr/space-GPT-image2-design) 的设计方法论。
> 当前仅做思路记录，后续自建图片技能时使用。

## 核心理念

**把"设计风格"编码成 AI 能准确理解的规则体系，而不是靠模糊的感觉描述。**

```
❌ "来个小清新风" → 太模糊，每个人理解不一样
✅ 底色 #FFF4D6 暖色 + 标题占画面 1/4~1/3 + ❌ 不要冷色
```

## 四维风格定义法

每个风格拆成4个维度：

### 1. 背景 (Background)
- 底色 hex / 是否有纹理 / 渐变方向
- 示例：`#FFF4D6` 暖色底，无纹理

### 2. 字体 (Typography)
- 标题字体 / 正文字体 / 字重 / 字距
- 示例：`Serif` 标题 / `Sans-Serif` 正文 / 标题 Bold / 字距 0.02em

### 3. 色板 (Color Palette)
- 5–8 个 hex 色，按角色分配
- 角色：背景 / 主文 / 次文 / 强调 / 装饰 / 分割线

### 4. 视觉元素 + Do/Don't
- 视觉元素清单（该风格必备/可选元素）
- 必须做的（Do）：如「标题带描边或 highlight 块」「数字用加粗大号」
- 绝对不做的（Don't）：如「❌ emoji 列表超过2个」「❌ 假数据或夸张话术」

| 风格 | 底色 | 色板关键词 | 适合场景 |
|------|------|-----------|---------|
| 小红书活力 (xhs-vibrant) | #FFF4D6 | 暖色+红色笔触 | 小红书封面 |

| 深色氛围 (dark-atmospheric) | 深色 | 紫青粉霓虹 | 科技/氛围感 |

## 类型规范系统

每种内容类型定义必填字段，缺一个追问一个——省10分钟描述，省半小时返工。

| 类型 | 必填字段 | 输出比例 |
|------|---------|---------|
| 小红书封面 | 主标题 / 副标题 / 视觉描述 / 主色倾向 | 3:4 |
| PPT配图 | 核心观点 / 关键数据 / 3-5要点 / 视觉锚点 | 16:9 |
| 公众号头图 | 标题 / 金句 / 主题色 / 氛围词 | 2.35:1 |
| 结构化图表 | 类型 / 节点 / 关系 / 强调项 | 自适 |
| 文章逻辑图 | 核心论点 / 逻辑结构 / 关键概念 | 自适 |

## 提示词模板结构

```
[1] 类型规范 — 比例 / 信息密度 / 必填字段
[2] 风格指令 — 完整 hex / 字体 / Do & Don't
[3] 用户内容 — 标题 / 节点 / 数据
[4] 通用约束 — 中文字符精确 / 无 logo 水印 / 留白足够
```

提示词控制在 ~1000 字符内，过长的会被 API 拒绝。

## 80分原则

> "这是一个 80 分的 skill，不是 100 分的产品。对不愿意打开 Photoshop / Figma 的人，80 分的 skill 比 100 分的产品好用。"

### 适用判断

| 场景 | 用这个思路 | 说明 |
|------|:---------:|------|
| 公众号头图 | ✅ | 我们还没有专用能力 |
| 小红书封面 | ✅ | 同样空白 |
| PPT配图 | ⚠️ | 我们已有 PPT 能力，可以等需要时再扩展 |
| 结构化图表 | ❌ | 已有 diagram-maker，不需要 PNG 出图 |
| 批量文章配图 | ❌ | 不是你的高频场景 |

## 后续落地

造图片技能时，直接复用这个框架：

1. 确定类型 → 用必填字段表确认输入完整性
2. 选风格 → 四维定义 + Do/Don't
3. 组装提示词 → [类型 + 风格 + 内容 + 约束] 四段模板
4. 出图 → 调用图像生成 API
