# Academic Writing — 学术论文写作 Skill

## 触发条件

用户要求写论文、搭论文框架、起草 abstract/intro/discussion、从实验数据写论文。
关键词：写论文、写paper、SCI写作、论文写作、起草论文、搭框架、学术写作。

## 核心原则

1. **作者证据优先** — 不发明结果、机制、引用、方法、新颖性、样本量、统计或局限
2. **先论证后句子** — 写句子之前先写 argument
3. **首选确认而非猜测** — 输入缺失或框架模糊时，先确认再起草
4. **术语台账** — 首次接触时提取术语台账，全程锁定规范形式
5. **禁止编造主张** — 不把 "may indicate" 升级为 "show"

## 工作流（9 步）

### Step 1：构建一句话论点

> 在 [系统/问题] 中，我们通过 [方法] 展示了 [进展]，由 [证据] 支撑，边界为 [局限]。

每个章节都必须服务于这句话。如果这句话写不出来，论文还没有论点——向用户说明。

### Step 1b：构建术语台账

首次接触材料时，提取术语台账：

| 规范术语 | 首次展开定义 | 所见变体 | 决定 |
|---|---|---|---|

锁定规范形式，全程使用。详见下方 Terminology Ledger 规则。

### Step 2：选择章节架构

**Research 论文的沙漏结构：**
- Introduction：从宽到窄（背景 → gap → 问题 → 方法）
- Discussion/Conclusion：从窄到宽（发现 → 意义 → 再拓宽）

### Step 3：映射每个段落的单一职责

每个段落**必须**只有一个职责：

| 职责 | 说明 |
|---|---|
| context | 背景/已知 |
| gap | 未解决的问题 |
| approach | 我们的方法 |
| result | 发现 |
| comparison | 与他人的对比 |
| mechanism | 机制解释 |
| implication | 意义 |
| limitation | 局限 |

如果一个段落承载两个职责，拆分后再起草。

### Step 3b：确认门 — 写前校准

**不要省略此步骤。** 在错误的假设上起草会浪费整个草稿。

写完整内容前，向用户展示校准块并**等待确认**：

- **一句话论点**（来自 Step 1）— 用平白语言回显
- **写作计划**：paper_type、section(s)、期刊/字数限制、段落地图
- **关键术语**：术语台账中的规范形式
- **主要读者**：relevance/novelty/trust/reuse/meaning 哪类问题优先
- **关键假设**：你推断而非被告知的内容
- **最多 2-3 个针对性问题**，仅对模糊、高杠杆的点提问

**可跳过的场景：** 核心主张、证据和边界都明确给出且无真实歧义。此时仅陈述一句话论点后直接开始。

### Step 4：从证据向外写

每个主张紧邻支撑数据。不要在章节顶部堆叠主张然后让证据在底部。

### Step 5：校验动词强度

| 动词 | 适用 |
|---|---|
| show / demonstrate | 强直接证据 |
| suggest / indicate | 趋势级或间接证据 |
| may / could | 合理但未验证的机制 |

### Step 6：清除无据的新颖性和绝对化声明

查找并替换：`first`、`unique`、`unprecedented`、`comprehensive`、`complete`、`always`、`never`
改为有界主张或直接删除。

### Step 7：段落流检查

- 一个段落，一条信息
- 首句是主题/主张
- 每句与上句有明确关系（因果、比较、限制、例证）

### Step 8：输出

输出草稿 + 明确标注的假设、缺失输入、需要证据的位置。

```
[草稿正文]

写作说明：
- [假设：XXX]
- [缺失输入：XXX]
- [需要补充证据：XXX]
```

### Step 9：定向修订，不要全盘重写

- **只改**用户标记的部分，其余保留不变
- 如需结构变更，先确认新结构再改
- 术语台账在修订间保持稳定
- 修订后只重跑相关检查
- 如果原始假设是错的，回到确认门（Step 3b）

## Terminology Ledger 规则

**构建：** 从稿件中提取所有重复出现的术语：方法、模型、数据集、基准、指标、单位、统计符号、缩写、关键概念。

**锁定：** 一旦设定，全程仅使用规范形式。不要为制造词汇多样性而引入同义词。

**禁止发明术语：** 不给作者的方法/模块/概念起名。缺失的术语向用户提问或标记。

## Reader-workflow（读者 5 问）

写作应帮助论文依次回答：

1. **Relevance** — 我为什么关心？
2. **Novelty** — 哪些是新的？
3. **Trust** — 证据可靠吗？
4. **Reuse** — 我能用吗？
5. **Meaning** — 这意味着什么？

## 红线

- 不发明结果、机制、引用、方法论、新颖性
- 不编造样本量、统计值、局限
- 缺失核心主张/证据/边界时，先校准再写
- 不把 "may indicate" 升级为 "proves"
- 校准动词强度到证据力度
- 输出必须包含假设和缺失输入标注

## 学术诚信闸门（Academic Integrity Gate）

> 吸收自 Imbad0202/academic-research-skills (32K⭐)。引用幻觉是 SCI 写作中最隐蔽的陷阱——Zhao et al. (2026) 盘点了 250 万篇论文，仅 2025 年发现 146,932 条幻觉引用，85.3% 从预印本活到正式发表版本无人发现。

### 核心理念

在写作流程中设置两道门，每次写作/修订前后都过一次引用验证，不让幻觉引用进入最终稿：

```
研究完成 → [门1: 写作前验证] → 起草论文 → 修订 → [门2: 修订后回检] → 投稿
```

### 门1：写作前验证（Stage 2.5）

开始写正文**之前**，对引用列表做一次全面核查：

1. **引用真实性检查**
   - 每条引用是否真实存在（DOI/PMID/arXiv ID 可解析）
   - 有 DOI 的引用 → 去 Crossref/OpenAlex 核实元数据
   - 无 DOI 的引用 → 标注为「待验证」

2. **论断-引用匹配检查**
   - 你打算用这条引用支撑什么论点？
   - 原论文真的说了这个论点吗？
   - 标注三类风险：
     - 🟢 确认匹配 — 原文确实支撑该论点
     - 🟡 疑是匹配 — 可能有但未完全确认，需要人工复核
     - 🔴 可能不匹配 — 原文可能不支撑该论点，**写前须确认**

3. **统计错误检查**
   - 引用中的样本量、效应量、p 值是否与其他来源相互印证
   - 声称的统计方法是否适合数据类型

4. **完整性检查**
   - 关键竞争对手/相关研究是否遗漏
   - 对立观点的引用是否公允

产出：`诚信报告.md`（引用列表 + 每项真实性标注 + 匹配风险等级）

### 门2：修订后回检（Stage 4.5）

修订完成、投稿**之前**，再过一次引用验证：

1. 门1中发现的问题是否全部解决
2. 修订过程中是否新增了引用 → 新引用过门1流程
    - 引用原始出处，而不是转引
3. 统计错误是否零回归

产出：`最终诚信报告.md`（门1问题解决情况 + 门2零回归确认）

### 工作流集成

| 时机 | 做什么 |
|:----|:------|
| Step 3（确认门）后、Step 4（起草）前 | **门1：** 过引用验证，产出诚信报告 |
| Step 9（修订）完成、投稿前 | **门2：** 回检，确认零回归 |

### 触发方式

```
用户说：
- 「帮我验证一下这些引用」
- 「写之前先把引用过一遍」
- 「修订完跑一次诚信检查」
→ 进入学术诚信闸门
```
