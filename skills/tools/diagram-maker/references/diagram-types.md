# 图类型专项指导

从用户意图判断图类型，再用对应规则提取信息和布局。

## 意图 → 图类型

| 用户意图 | 图类型 | 关键词 |
|----------|--------|--------|
| 流程、步骤、操作顺序 | **流程图 Flowchart** | 流程、步骤、操作、workflow |
| 概念层级、头脑风暴 | **思维导图 Mind Map** | 脑图、概念、分类、mind map |
| 连接、依赖、关联 | **关系图 Relationship** | 关系、依赖、结构、关联 |
| 系统组件、架构 | **架构图 Architecture** | 架构、组件、模块、系统 |
| 数据流转、输入输出 | **数据流图 DFD** | 数据流、数据加工、data flow |
| 跨角色流程、职责划分 | **泳道图 Swimlane** | 跨部门、角色、职责、swimlane |
| 类结构、OOP设计 | **类图 Class Diagram** | 类、继承、OOP、对象 |
| 对象交互、消息顺序 | **序列图 Sequence** | 交互、信息、时序、sequence |
| 数据库设计、实体关系 | **ER图 ER Diagram** | 数据库、实体、表、数据模型 |

---

## 流程图 Flowchart

**提取：**
- 步骤列表（有顺序）
- 判断点（是/否分支）
- 起点和终点

**布局：** 从上到下，判断点用菱形，步骤用矩形。

**Excalidraw 元素：**
- 步骤 → `rectangle`，背景 `#a5d8ff`（浅蓝）
- 判断 → `diamond`，背景 `#ffd43b`（黄）
- 起点/终点 → 圆角矩形 `#b2f2bb`（浅绿）
- 箭头连接

---

## 思维导图 Mind Map

**提取：**
- 中心主题
- 主分支（3-6条）
- 子分支（每分支2-4条）

**布局：** 径向（中心向四周发散）

**公式：**
```
const angle = (2 * Math.PI * index) / branchCount;
const x = centerX + radius * Math.cos(angle);
const y = centerY + radius * Math.sin(angle);
```

**Excalidraw 元素：**
- 中心 → 大椭圆，背景 `#ffd43b`（黄）
- 分支 → 矩形，背景 `#a5d8ff`（蓝）
- 子分支 → 小矩形，背景 `#d0bfff`（紫）

---

## 关系图 Relationship

**提取：**
- 实体/节点（名称 + 描述）
- 关系（从→到，带标签）

**布局：** 网格或径向。

**网格公式：**
```
const columns = Math.ceil(Math.sqrt(entityCount));
const x = startX + (index % columns) * horizontalGap;
const y = startY + Math.floor(index / columns) * verticalGap;
```

**Excalidraw 元素：**
- 实体 → `rectangle`，背景 `#e9ecef`（浅灰）
- 关系 → `arrow`，带标签文字

---

## 架构图 Architecture

**提取：**
- 层级（用户层/业务层/数据层等）
- 组件（每层的模块）
- 连接方式

**布局：** 自下而上分层堆叠，或左到右。

**Excalidraw 元素：**
- 分层区域 → 大矩形（虚线边框，背景透明）
- 组件 → 小矩形，按层着色
- 箭头标注调用/依赖方向

**建议的层级色：**
- 用户层 → #a5d8ff
- 业务层 → #b2f2bb
- 数据层 → #ffc9c9
- 基础设施 → #d0bfff

---

## 数据流图 DFD

**提取：**
- 外部实体（数据来源/目的地）
- 处理过程（数据变换）
- 数据存储（数据库/文件）
- 数据流（箭头方向）

**注意：** DFD画的是**数据流动**，不是处理顺序。

**布局：** 从左到右或从左上到右下。

**Excalidraw 元素：**
- 外部实体 → `rectangle`，背景 `#e9ecef`
- 处理过程 → 圆角矩形（`roundness.type: 1`），背景 `#a5d8ff`
- 数据存储 → 一侧开口矩形（用两条线模拟），背景 `#d0bfff`
- 数据流 → `arrow`，带标签

---

## 泳道图 Swimlane

**提取：**
- 角色/部门（横排表头）
- 泳道（每个角色下方空间）
- 活动（泳道内的操作框）
- 流转箭头（含跨泳道传递）

**布局：** 表头在上，每个泳道纵向排列活动。

**Excalidraw 元素：**
- 角色表头 → 顶部矩形，背景按角色区分
- 分隔线 → 垂直虚线
- 活动 → 矩形，背景 `#e9ecef`
- 箭头连接，跨泳道箭头标注交接

---

## 类图 Class Diagram

**提取：**
- 类名
- 属性（可见性+名称+类型）
- 方法（可见性+返回值+参数）
- 关系（继承/实现/关联/依赖/聚合/组合）
- 多重性标注（1, 0..*, *等）

**可见性符号：** `+` public, `-` private, `#` protected

**布局：** 继承从上到下，关联左右连接。

**Excalidraw 元素：**
- 类 → 三层矩形（类名区/属性区/方法区），用水平线分隔
- 继承 → 实线+空心三角箭头
- 实现 → 虚线+空心三角箭头
- 关联 → 实线
- 多重性 → 在连线两端标注文字

---

## 序列图 Sequence

**提取：**
- 对象/参与者（水平排列在最上方）
- 生命线（每个对象向下的垂直线）
- 消息（水平箭头在生命线之间）
- 激活框（执行期间的纵向矩形）
- 时间顺序从上到下

**布局：** 对象在顶部横向排开，消息从上往下。

**Excalidraw 元素：**
- 对象 → 顶部矩形，背景 `#a5d8ff`
- 生命线 → 虚线（`dashed: true`），从对象底部向下
- 同步消息 → 实线箭头 → `#2f9e44`
- 异步消息 → 虚线箭头 `dashed: true` → `#e8590c`
- 返回消息 → 虚线箭头（回的方向）→ `#868e96`
- 激活框 → 细长矩形覆盖在生命线上，背景 `#e9ecef`

---

## ER图 ER Diagram

**提取：**
- 实体（矩形 + 名称）
- 属性（列在实体内部）
- 主键（标注 PK）
- 外键（标注 FK）
- 关系（连线）
- 基数：1:1 / 1:N / N:M

**布局：** 关系紧密的实体靠近摆放。

**Excalidraw 元素：**
- 实体 → `rectangle`，背景 `#a5d8ff`
- 关联实体（N:M 关系表）→ 虚线边框矩形
- 属性文字居中排在实体矩形内
- 关系连线 → `arrow`，两端标注基数
- PK 标注 → 下划线文字
- FK 标注 → 斜体文字
