# 复杂度管理与验证

## 元素数量建议

| 图类型 | 推荐数量 | 上限 |
|--------|---------|:----:|
| 流程图步骤 | 3-10 | 15 |
| 关系图实体 | 3-8 | 12 |
| 思维导图分支 | 4-6 | 8 |
| 思维导图子分支/每支 | 2-4 | 6 |
| 架构图组件 | 4-8 | 12 |

## 复杂度管理策略

**超过上限时：**

1. 先口头提示："有 X 个元素，建议拆成 Y 张图"
2. 第一张图画高层的总览（5-9个核心元素）
3. 后续图逐个展开子系统/子流程
4. 不等确认，直接给出分拆建议

**示例：**
```
有15个组件，建议拆成2张图：
1. 高层架构（7个核心模块的概览）
2. 模块A内部详细结构（8个子组件）
先从高层开始？
```

## 颜色方案

| 用途 | 颜色 | 十六进制 |
|------|------|:--------:|
| 主要元素 / 操作 | 浅蓝 | `#a5d8ff` |
| 次要元素 / 数据 | 浅绿 | `#b2f2bb` |
| 核心 / 中心节点 | 黄色 | `#ffd43b` |
| 警告 / 风险 | 浅红 | `#ffc9c9` |
| 外部实体 | 浅灰 | `#e9ecef` |
| 分支 / 子节点 | 浅紫 | `#d0bfff` |
| 文字 | 黑色 | `#000000` |
| 画布背景 | 白色 | `#ffffff` |

## 布局间距

- 水平间距：200-300px 之间
- 垂直间距：100-150px 之间
- 最小标注形状：120x60px
- 正文字体：16-20px

## 验证 Checklist

输出前逐项检查：

- [ ] 所有元素有唯一 ID（可用时间戳+随机后缀）
- [ ] 坐标不重叠（水平间距≥200, 垂直间距≥100）
- [ ] 文字可读（≥16px）
- [ ] 所有 text 元素用 `fontFamily: 5`（Excalifont）
- [ ] 箭头逻辑正确（谁连谁，方向对不对）
- [ ] 颜色一致（按上表，不乱用颜色）
- [ ] 文件是合法 JSON
- [ ] 元素数量合理（不超过上限）
- [ ] 复杂图已拆分为多张（超过上限时）
- [ ] 连线在节点后方（先画线，再画节点）
