# Learning Record 模板

> 来源：Matt Pocock `/teach` — LEARNING-RECORD-FORMAT.md

Learning records 放在 `./learning-records/` 中，使用顺序编号：`0001-slug.md`, `0002-slug.md` 等。目录按需创建 — 有第一条记录时才建。

它们是教学版的 ADR（架构决策记录）：捕获非显而易见的洞见、关键洞察、和已声明的已有知识，用于计算最近发展区。

## 模板

```md
# {简短标题：学到了什么或确立了什么}

{1-3 句话：学到了什么（或确立了什么已有知识），以及为什么这对未来的session重要。}
```

以上就是完整格式。一条 learning record 可以只有一个段落。价值在于记录「这个现在已经知道了」和「为什么这会改变接下来要教的内容」 — 不在填满各个section。

## 可选 section

只有确实有增补价值时才包含。大多数记录不需要这些。

- **Status** frontmatter (`active | superseded by LR-NNNN`) — 当之前的理解被证明是错的且被替换时使用
- **Evidence** — 用户如何证明了这个理解（回答对了一个问题、完成了一个练习、引用了既往经验）。在可能需要回溯验证这条说法时有价值
- **Implications** — 这条洞见解锁了什么或排除了什么未来session的可能性。不显而易见的关联值得记录

## 编号

扫描 `./learning-records/` 找到最高编号 +1。

## 什么时候写一条 learning record

以下任一为真时写：

1. **用户对非平凡的东西展示了真正的理解** — 不只是暴露了知识，而是有证据表明他们能正确运用这个概念。这设置了下一步教什么的新基准
2. **用户透露了已有知识** — "我已经会X"。记录下来以免未来的session重复教。同时记录声称的深度
3. **纠正了一个误解** — 用户之前相信了错误的、现在明白了为什么错。这些高价值：它们预测了相关主题未来的绊脚石
4. **Mission 因学习成果而移动了** — 用户发现自己在乎的和之前以为的不同。交叉链接到 [[MISSION.md]] 并更新它

### 什么不算

- 仅仅覆盖了材料。覆盖不是学习。等证据
- 已在 [[GLOSSARY.md]] 中简洁记录为术语定义的任何内容。不要重复
- Session到session的活动日志。Learning records 不是日记 — 它们是决策级别的洞察

## 替代

当后来的记录与之前的记录矛盾时（用户的理解加深或更正了），标记旧记录 `Status: superseded by LR-NNNN` 而不是删除它。理解如何演变的历史本身就是有用的信号
