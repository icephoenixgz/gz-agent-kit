# RESOURCES.md 模板

> 来源：Matt Pocock `/teach` — RESOURCES-FORMAT.md

`RESOURCES.md` 是此主题的精选可信资源清单。讲解的知识应从这里提取，而非参数猜测。智慧来自此处列出的社群。

## 结构

```md
# {主题} 资源

## Knowledge（知识）

- [书籍: 《……》 — 作者名](https://example.com)
  覆盖范围/适用场景：……

- [文章: "……" — 作者名（来源平台）](https://example.com)
  覆盖范围/适用场景：……

## Wisdom（智慧 / 社群）

- [r/XXX](https://reddit.com/r/XXX)
  高信噪比子论坛，强审核质量。用途：……
- 本地：{城市的某线下活动/课程}
  用途：实时的反馈和指导
```

## 规则

- **只收录高信赖源**。优先一手来源、公认专家、同行评议的工作、有强管理的社群。如果某资源是穿着教育外衣的市场营销，排除它
- **为每个条目做注解**。裸链接三个月后毫无用处。加一行：覆盖什么、何时用
- **按 Knowledge / Wisdom 分组**。镜像 [SKILL.md](./SKILL.md) 中的哲学。一个资源只出现在一组里完全OK
- **明确标识缺口**。如果 Mission 需要但不存在好的资源，写一个 `## Gaps` section 列出缺失什么。这驱动未来的搜索
- **无情修剪**。一个被发现是错的、肤浅的、或偏离 Mission 的资源应该被移除，而非埋葬。五个锋利的资源好过三十个平庸的
- **记录社群偏好**。如果用户选择了不加入社群，在这里注明，以免未来的session持续提议
