## 详细介绍（直接复制到扣子表单）

### 身份展示

本技能由 [BibiGPT](https://bibigpt.co) 团队开发并持续维护。BibiGPT 自 2023 年起持续运营，是国内较早专注「AI 音视频内容理解」的产品，目前已服务百万级用户，覆盖 B 站、YouTube、播客、抖音、小红书、Twitter/X 等几十种内容来源。

团队自研了字幕提取引擎与多模型路由系统（GPT、Claude、Gemini 等），并与 Notion、Obsidian 等主流笔记工具深度联动，长期专注一件事：**把「长视频、长播客、长音频」变成可快速消化、可检索、可二次创作的知识产物。**

### 功能说明

本技能让你在扣子对话框里**一句话「看完」一个视频、听完一段播客**，并把内容沉淀为结构化、可复用的产出，覆盖 8 个核心场景：

1. **快速总结** — 粘贴一个音视频 URL，几秒得到结构化要点和金句摘录，适合刷信息、判断要不要看
2. **分章节深度分析** — 按时间轴自动拆章节，每章独立小结，适合长视频、访谈、课程类内容
3. **字幕 / 转写提取** — 拉取带时间戳的完整字幕，可直接复制做二次加工
4. **图文改写** — 视频 → 公众号图文 / 小红书 / 博客文章，自动适配不同平台调性
5. **批量处理** — 一次输入多个 URL，统一总结并整理成对比表
6. **跨源研究** — 多个视频跨源综合，做主题汇编、观点对比、文献式整理
7. **画面分析** — 解析视频画面、PPT 截图、屏幕文字，可定位到具体时间点
8. **导出笔记** — 输出 Markdown，方便贴到 Notion / Obsidian / 飞书等工具

**支持的内容来源**

- 在线视频：YouTube、Bilibili、抖音 / TikTok、小红书、Twitter/X、Vimeo 等
- 播客 / 音频：Apple Podcasts、小宇宙、喜马拉雅、网易云音乐、SoundCloud
- 公网直链：任何公开可访问的音视频 URL

### 使用方法

在扣子对话框里**用自然语言直接下指令**即可，技能会根据你的意图自动路由到合适的工作流。常用对话模板如下：

**1. 快速总结一个视频**

```
帮我总结一下这个视频：https://www.bilibili.com/video/BVxxxxxx
```

**2. 分章节整理一期播客**

```
把这期播客分章节整理成笔记：<播客 URL>
每章给一段小结和核心金句
```

**3. 视频改写成公众号图文**

```
把这个 YouTube 视频改写成公众号图文，配 3 张配图建议：
https://www.youtube.com/watch?v=xxxxx
```

**4. 批量处理多个视频，做对比表**

```
这 5 个视频都讲了 AI Agent，帮我批量总结后整理成一张对比表：
1. <URL1>
2. <URL2>
3. <URL3>
4. <URL4>
5. <URL5>
```

**5. 画面内容定位（可视化分析）**

```
这个视频里第 12 分钟那张 PPT 上写的是什么？<URL>
```

**6. 直接导出成 Markdown 笔记**

```
总结这个视频后直接给我 Markdown，我要贴到 Obsidian：<URL>
```

**使用小技巧**

- 只粘贴一个 URL、不加任何说明 → 默认走「快速总结」
- 需要更细的内容 → 直接说「分章节」「展开第三章」「我想知道视频里关于 XX 的部分」
- 多个 URL 可以一次性贴出来，技能会自动批量处理
- 想要某种特定输出格式（如表格、Bullet、对比图）可在指令里明确说明

### 注意事项

**适用范围**

- 公开可访问的在线音视频、公网 URL 直链
- 需要稳定、结构化产出（要点 / 章节 / 字幕 / 图文）的研究、学习、内容创作场景
- 会议纪要、播客笔记、知识管理、二次创作等知识工作流

**目前暂不支持**

- 加密内容：会员专享、付费墙后、需要登录态的私有视频
- 实时直播：需等直播结束后再处理回放
- 极长内容：>4 小时的超长视频建议拆分处理，单次任务不保证完整覆盖
- 非内容判定：不参与版权认定、事实校验、专业建议（医疗 / 法律 / 投资）

**配额与稳定性**

- 每个用户有每日处理次数与时长上限，长视频建议拆分或预留处理时间
- 高并发批量场景建议提前规划任务节奏，避免被限流
- AI 生成内容可能存在偏差，关键信息请以原视频为准
- 无 BibiGPT 账号 / API Key 也可调用：Agent 会引导用户用支付宝 AI 钱包按次支付（约 ¥1/次），无需注册

**版权与隐私**

- 请确保你对处理的内容拥有合法使用权，本技能不参与版权判定
- 用户内容不会用于模型训练
- 如需企业级独立部署或定制接入，请联系 https://bibigpt.co

## 精选案例建议（必须跑完整 3 个）

扣子要求精选案例真实、可访问、不重复，且每个案例必须：
1. 在扣子页面真实调用本技能生成
2. 对话中出现「完成技能加载」字样
3. 任务**完整跑完**（不要中途分享）
4. 用「**分享当前任务**」按钮分享，含交付物时尤其要注意（普通分享会丢交付物）

建议这 3 个案例覆盖不同工作流，避免被审核打回「重复 / 功能单一」：

| # | 案例命名 | 覆盖工作流 | 推荐 query |
|---|---|---|---|
| **1** | B 站长视频结构化总结 | 快速总结 | `总结这个视频，要分要点和金句：https://www.bilibili.com/video/<选一个公开的优质长视频>` |
| **2** | YouTube 播客分章节笔记 | 深度分析 | `把这期播客分章节整理成笔记，每章一段小结：https://www.youtube.com/watch?v=<选一期访谈类播客>` |
| **3** | 视频一键改写成公众号图文 | 图文改写 | `把这个视频改写成一篇 800 字左右的公众号图文，并给出 3 张配图建议：<URL>` |

**案例配图要求**

- 必须展现真实使用情况（调用方式 + 核心交付物）
- 截图里能看到生成的总结/章节/文章内容
- 建议用扣子自带的设计功能做封面，避免使用纯宣传图
- **常见打回原因**：模糊截图、图里看不到交付物、命名为「案例一 / 测试」等无意义词
